カスタムsteeringの作成対象を特定し、テンプレートの有無を確認せよ。

**やること:**
1. ユーザーの入力からカスタムsteeringのドメイン/トピックを特定する
2. `.takt/knowledge/steering-custom-template-files/` に対応するテンプレートが存在するか確認する
3. `.kiro/steering/` に同名のファイルが既に存在するか確認する
4. 判定結果をレポートする

**利用可能なテンプレート:**

| テンプレート | ドメイン |
|-------------|---------|
| architecture.md | アーキテクチャスタイル、レイヤー境界、依存ルール |
| api-standards.md | REST/GraphQL規約、エラーハンドリング |
| testing.md | テスト構成、モッキング、カバレッジ |
| security.md | 認証パターン、入力検証、シークレット管理 |
| database.md | スキーマ設計、マイグレーション、クエリパターン |
| error-handling.md | エラー型、ロギング、リトライ戦略 |
| authentication.md | 認証フロー、権限管理、セッション管理 |
| deployment.md | CI/CD、環境構成、ロールバック手順 |

**判定基準:**

| 条件 | モード |
|------|--------|
| テンプレートあり、既存ファイルなし | テンプレートベース生成 |
| テンプレートなし、既存ファイルなし | スクラッチ生成 |
| 既存ファイルあり | 更新 |

**レポートに含める情報:**
- 対象トピック名
- テンプレートの有無
- 既存ファイルの有無
- 推奨されるモード
